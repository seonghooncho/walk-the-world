# 실서비스 QA 리포트

## 근본 목적

운영 URL에서 실제 브라우저와 실제 API를 사용해 mock 테스트로 놓칠 수 있는 배포 누락, 경계값 미적용, 권한 누락을 조기에 발견한다.

## 비목적

외부 소셜 제공자의 실제 계정 선택 UI 자동화, 운영 사용자 데이터 대상 파괴 테스트, 인프라 구조 재설계는 이 리포트 범위에 포함하지 않는다.

## 실행 기준

- 대상 URL: `https://walk2world.cloud`
- 실행 명령: `cd fe && npm run e2e:live`
- 테스트 데이터: `ww-live-*-${RUN_ID}@example.com` 형식의 일회성 계정
- 정리 정책: 생성 게시글 삭제 후 테스트 계정 회원탈퇴 시도

| 입력 경계 | 501자 댓글 | 실패 | `400 INVALID_REQUEST`가 아니라 `201` 생성 |
| 페이지네이션 | 게시글 `limit=0` | 실패 | meta `limit=1`이 아니라 `0` 반환 |
| 친구 | 알 수 없는 친구 추가 method | 실패 | `400 INVALID_FRIEND_METHOD`가 아니라 `500 INTERNAL_ERROR` |
| 채팅 권한 | 비참여자 메시지 조회 | 실패 | `403 CHAT403`이 아니라 `200` 반환 |
| 채팅 입력 | 501자 메시지 | 실패 | `400 INVALID_REQUEST`가 아니라 `201` 생성 |
| 채팅 재진입 | 500자 메시지 전송 후 채팅방 재진입 | 통과 | 스레드에 메시지 유지 |
| 회원탈퇴 | `DELETE /api/users/v1/me` | 실패 | `200`이 아니라 `500 INTERNAL_ERROR` |

## 수정 기준

- `V3__add_user_withdrawal_state.sql`가 운영 DB에 적용되어야 한다.
- 백엔드 Lambda가 경계값 PR 이후 artifact로 갱신되어야 한다.
- 배포 후 `npm run e2e:live`는 공개/인증/입력/권한/탈퇴 시나리오를 모두 통과해야 한다.
