# walk2world 제품 요구사항

## 근본 목적

사용자가 걸음 수로 도시를 열고, 현재 도시에서 미션과 사람을 만나며, 다음 도시까지의 진행감을 끊기지 않고 이해하게 하는 것이 목적이다.

## 비목적

이번 문서는 백엔드 DB 정책이나 운영 인프라 설계를 상세화하지 않는다. 모바일 앱의 네이티브 동작도 웹 기준이 안정화된 뒤 별도 문서에서 다룬다.

## 웹 핵심 경험

- `/` 홈은 현재 도시, 누적 걸음 수, 다음 도시까지 남은 걸음, 진행 중인 미션, 도시 커뮤니티 진입을 첫 화면에서 보여준다.
- `/map` 노선도는 현재 체크포인트, 다음 도시, 전체 방문 수, 완료율, 도시별 잠금/해금 상태를 보여준다.
- `/demo`는 미로그인 사용자가 도쿄 preview, 미션, 도시 커뮤니티 샘플을 체험하는 화면이다.
- `/city`는 현재 도시의 게시물과 멤버를 보여주고, 게시물이 없으면 첫 게시물 작성 행동을 제안한다.
- `/feed`는 친구 채팅방 목록을 보여주고, 채팅방이 없으면 도시 커뮤니티로 이동해 친구를 추가하게 한다.
- `/profile`은 나의 여행 통계, 현재 도시, QR 친구 추가, 배지와 설정 진입을 제공한다.

## 로그인 경계

- 공개 preview: `/`, `/map`, `/demo`, `/login`, `/terms`, `/privacy`, `/auth/callback`.
- 로그인 필요: `/city`, `/feed`, `/chat/:chatId`, `/post/:postId`, `/profile`, `/profile/badges`, `/add-friend/:userId`.
- 보호 화면 접근 시 `/login?redirect=<원래 경로>`로 이동하고, 로그인 성공 후 redirect 경로로 복귀한다.
- OAuth callback은 hash 또는 query로 전달된 토큰을 저장할 수 있어야 하며, error query는 로그인 화면으로 복귀시킨다.

## 오류 처리 기준

- access token이 401을 받으면 refresh token으로 한 번 재시도한다.
- refresh가 실패한 경우에만 토큰을 삭제하고 로그인으로 유도한다.
- 프로필 조회의 일시적인 네트워크 오류는 곧바로 토큰 삭제로 처리하지 않는다.
- API 데이터가 비어 있는 화면은 빈 상태와 복구 행동을 보여준다.
